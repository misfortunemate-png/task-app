## Phase 3に持ち越す事項

### 確定済みの設計方針

- **データ同期**: Firestoreリアルタイム同期（onSnapshot）。クライアント直接接続、サーバーレス
- **ソート・フィルタ**: クライアント側で実施。Firestoreからはuidフィルタのみ。複合インデックスは使用しない。データ量（個人利用、数百件規模）に対して十分な性能
- **セキュリティ**: uidベースのセキュリティルール。ショウゴさんのGoogleアカウントのみアクセス可
- **無料枠**: Spark プラン。日次50K読み/20K書きに対し、実使用は数十〜数百回。課金リスクなし
- **仕様書の粒度**: 「何を」は仕様書に書く。「どのように」はPGに裁量を残す

### Phase 3の主要機能（企画ノートより）

- 担当キャラ選択（ハコネ🌸 / クリーデ📘 / フラン🛸）
- セリフ表示システム（モーダル形式を推奨）
- 4つのライフサイクルイベント（登録・達成・放置・キャンセル）
- カテゴリ別セリフの優先ロジック（専用あれば優先、なければgenericから抽選）
- セリフデータのインポート/エクスポート（JSON）
- 表情システム（emoji、将来イラスト差し替え可）
- 初期セリフデータの同梱

### 未決定事項（Phase 3で決める）

- デフォルト担当キャラの挙動（毎回選ぶ / 前回記憶 / 固定デフォルト）
- 放置判定しきい値（案: 24時間）と再催促間隔
- ハコネ・フランのセリフをPMが代筆するか、各プロジェクトに正式発注するか
- タスクカードへのキャラテーマカラー反映の程度
- モバイルレイアウトの調整
- Google Fonts（Klee One）の導入可否

### 既知の不具合

- タスク追加フォームのモバイル表示に軽微なずれ
