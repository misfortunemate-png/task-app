## 5. キャラ選択・キャラタブ（§3）

### 5.1 Firestoreスキーマ変更

taskドキュメントに以下のフィールドを追加する。

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| character | string | はい | キャラ定義のid値（"hakone" / "creede" / "fran"） |
| dueDate | timestamp or null | いいえ | 完了予定日。nullの場合は放置判定なし |
| dueDateUnlocked | boolean | いいえ | なでなでによる期限延長ロック解除フラグ。デフォルトfalse |

### 5.2 Firestoreセキュリティルール

既存のuidベースルールに、新フィールドの書き込みを許可する。バリデーションルール:

- `character` は文字列であること
- `dueDate` はtimestampまたはnullであること
- `dueDateUnlocked` はbooleanであること

### 5.3 キャラ選択UI（タスク登録フォーム）

- タスク登録フォーム内に担当キャラ選択を追加
- キャラ定義配列から動的に選択肢を生成
- 各選択肢にemoji + 名前を表示
- 必須選択（未選択での登録は不可。バリデーションで阻止）

### 5.4 キャラタブ（タスク画面の2層目）

- 「全員」タブ + キャラ定義配列から動的にタブを生成
- タブ選択時の動作:
  1. タスクリストを `character` フィールドでフィルタ（「全員」はフィルタなし）
  2. CSS変数を選択キャラの `color` / `colorBg` / `colorAccent` に切り替え
  3. コンテンツ領域にキャラのemoji（大表示）を表示
- 「全員」タブのテーマはニュートラル（Phase 2のデフォルトカラー）。三人のemojiを並べて表示
- タスクカードに担当キャラのアクセントカラーを反映（左ボーダー等。PGの裁量で）

## 6. セリフ表示システム（§4）

### 6.1 セリフデータ構造

```json
{
  "character": "hakone",
  "category": "generic",
  "event": "register",
  "lines": [
    { "text": "セリフテキスト", "face": "表情キー" }
  ]
}
```

- `category`: "generic" は汎用セリフ。カテゴリIDを指定するとカテゴリ専用セリフ
- `event`: "register" / "complete" / "neglect" / "cancel"
- `lines`: 配列。この中からランダムに1つ選択

### 6.2 セリフ選択ロジック

1. 該当キャラ × 該当イベント × 該当カテゴリ のセリフセットを検索
2. 見つかった場合: そのlines[]からランダムに1つ選択
3. 見つからない場合: 該当キャラ × 該当イベント × "generic" のlines[]からランダムに1つ選択
4. genericも見つからない場合: フォールバック `{ "text": "……。", "face": "normal" }` を表示

### 6.3 セリフモーダル

- 表示タイミング: 登録完了時 / 完了操作時 / キャンセル操作時 / 放置判定時
- 表示内容: キャラemoji（大表示）+ 表情emoji + セリフテキスト + キャラ名
- 閉じ方: モーダル外タップ、または閉じるボタン
- フォント: セリフテキストにKlee Oneを適用

### 6.4 表情マッピング

| キー | emoji | キー | emoji |
|------|-------|------|-------|
| normal | 😊 | dere | 🥰 |
| smile | 😆 | mischief | 😏 |
| blush | 😳 | salute | 🫡 |
| angry | 😤 | troubled | 😑 |
| sad | 😢 | warning | ⚠️ |
| surprise | 😲 | stare | 🫤 |

将来イラスト画像に差し替え可能なように、表情キーから表示要素への変換を一箇所に集約すること。

## 8. 放置判定・なでなで（§6）

### 8.1 放置判定

- 判定タイミング: アプリ起動時（コンポーネントのマウント時）
- 判定条件: `dueDate` が現在日時より前 かつ ステータスが未完了
- 該当タスクが複数ある場合: `dueDate` が最も古い1件のみモーダル表示
- 同一タスクの連続表示を避けるため、最後に表示したタスクIDをsessionStorageに保持し、次の起動では別のタスクを表示する。全タスクを一巡したらリセット

### 8.2 期限延長の制限

- タスク登録後、完了予定日を**後ろに延長する**編集は通常時不可
  - 「編集」画面で日付を変更しようとした際、元の日付より後の日付は選択不可にする
  - 前倒し（元の日付より前への変更）は常に可能
- Firestoreの `dueDateUnlocked` が `true` の場合のみ、後方延長が可能になる
- 延長後、`dueDateUnlocked` は `false` にリセットする（次回の延長には再度なでなでが必要）

### 8.3 なでなで

- 放置判定モーダル上にキャラemoji（大表示）を配置
- emojiをタップするたびに以下を実行:
  1. タップカウンターをインクリメント
  2. セリフを段階的に変化させる
  3. しきい値に達したら解除セリフを表示し、Firestoreの `dueDateUnlocked` を `true` に更新
- しきい値: デフォルト5回（設定画面で変更可能）
- なでなでセリフ（キャラごと3パターン: 序盤 / 途中 / 解除時）:

```json
{
  "hakone": {
    "early": { "text": "ん……？ なに？", "face": "surprise" },
    "mid":   { "text": "もぉ〜、なでなでしたって許さないんだから……", "face": "blush" },
    "unlock":{ "text": "……しょうがないなぁ♡ 今回だけだよ？", "face": "dere" }
  },
  "creede": {
    "early": { "text": "あの、お仕事中なのですが……", "face": "troubled" },
    "mid":   { "text": "ショウゴさん、これで解決するとは思わないでくださいね？", "face": "stare" },
    "unlock":{ "text": "……もう。帳簿に「期限延長（情状酌量）」と記入しますからね。", "face": "mischief" }
  },
  "fran": {
    "early": { "text": "接触を検知。意図を解析中……", "face": "normal" },
    "mid":   { "text": "繰り返しの接触を確認。マスター、これは交渉ですか？", "face": "surprise" },
    "unlock":{ "text": "……了解。期限延長を許可します。マスター命令には逆らえません。", "face": "blush" }

```

- 「序盤」は1回目〜(しきい値/2)回目、「途中」は(しきい値/2)+1回目〜しきい値-1回目、「解除」はしきい値到達時に表示
- デバッグモードON時: タップカウンター（現在N/しきい値）を画面に表示

## 9. 設定画面・デバッグモード（§7）

### 9.1 設定画面

設定タブから遷移する画面。各設定値はlocalStorageに保存。

| 設定項目 | キー | 型 | デフォルト値 |
|----------|------|----|-------------|
| 起動時のキャラタブ | defaultCharTab | string | "all" |
| なでなで解除回数 | nadeThreshold | number | 5 |
| デバッグモード | debugMode | boolean | false |

### 9.2 デバッグモード

設定画面でONにすると、タスク画面の下部（または設定画面内）にデバッグパネルが表示される。

| 機能 | 操作 |
|------|------|
| セリフ強制表示 | キャラ・イベント・カテゴリを選択して「表示」ボタン → セリフモーダルが開く |
| 放置判定トリガー | 「放置判定を実行」ボタン → 日付条件を無視して放置判定を走らせる |
| なでなでカウンター | 放置モーダル上に「N / しきい値」を常時表示 |
| セリフデータ一覧 | バンドルされている全セリフをリスト表示 |
| キャラパラメータ変更 | 各パラメータをスライダーまたは数値入力で一時変更。リロードでリセット |

## 10. エラーハンドリング（§8）

### 10.1 トースト通知

- 画面下部に一時的なメッセージを表示する共通コンポーネント
- 表示時間: 3秒後に自動消去
- 種類: success（緑）/ error（赤）/ info（グレー）

### 10.2 エラーハンドリング対象

| エラー | 対応 |
|--------|------|
| Firestore書き込み失敗 | トースト（error）「保存に失敗しました。もう一度お試しください」 |
| Firestore読み取り失敗 | トースト（error）「データの読み込みに失敗しました」 |
| セリフデータ不整合 | フォールバックセリフを表示。コンソールにwarning出力 |
| 認証切れ | ログイン画面への遷移を促すトースト（info） |
| 完了予定日バリデーション | 登録フォーム内にインラインエラー「過去の日付は設定できません」 |
| キャラ未選択バリデーション | 登録フォーム内にインラインエラー「担当キャラを選択してください」 |
