## 1. 概要

- 何を作るか: Phase 3のキャラクターシステムの上に、遊び・愛着・運用効率の層を追加する。メニューを上部集約しPixel 10に最適化、プレースホルダー画面（キャラ・ガチャ）を実画面に置換、セリフシステムを拡張、Claude→モバイルのタスク登録フローを整備する
- なぜ作るか: タスク管理を「楽しい」から「愛着がわく・使い続けたくなる」に進化させる
- 誰が使うか: ショウゴ（個人利用）。主にPixel 10のモバイルChromeで操作

## 3. 技術選定

| 技術 | 理由 |
|---|---|
| motion (旧Framer Motion) | つんつんのwhileTapアニメーション、モーダル入退場、ガチャカプセルバウンス。宣言的APIによりPGの試行錯誤コスト削減 |
| canvas-confetti | ガチャ排出演出、レアセリフ★4以上の特別エフェクト。shapeFromTextでキャラemojiの紙吹雪が可能。依存なし、6KB |
| IndexedDB (raw API) | ギャラリー画像のローカル保存。Firestoreを使わずブラウザ内に閉じる。prompt-vault-v2で実証済み |
| Clipboard API | ご褒美プロンプトのコピー。フォールバックとしてexecCommand('copy') |

## 4. 機能仕様

### §B0: メニュー上部集約

**何を**: Phase 3の下部固定メインメニューを上部固定に移動し、2層構造にする。

**1層目（メインナビ）**: 5タブ横並び。アイコン＋ラベル。高さ約32px。`position: sticky; top: 0`で上部固定。

**2層目（サブナビ）**: 1層目の選択に応じて内容が切り替わる汎用コンポーネント。高さ約30px。1層目の直下に配置。

| 1層目 | 2層目の内容 |
|---|---|
| タスク | 全員 / 🌸ハコネ / 📘クリーデ / 🛸フラン（キャラフィルタ、既存CharTabの移植）|
| 通知 | なし |
| キャラ | 🌸ハコネ / 📘クリーデ / 🛸フラン / 📊（ランキング）|
| ガチャ | なし |
| 設定 | なし |

**制約**: 1層目＋2層目の合計高さを65px以内に抑えること。Pixel 10のviewport（412×750〜900px）において、コンテンツ領域を十分確保するため。

### §B1: ライブラリ導入

**何を**: `motion`と`canvas-confetti`をnpm依存に追加する。

```
npm install motion canvas-confetti
```

motionはReact 19との互換性を確認してからバージョンを決定すること。

### §B2: データ基盤

**セリフデータ構造の拡張**:

既存のlines.jsの各セリフオブジェクトに以下のオプションフィールドを追加する。

```js
{
  text: "セリフ本文",
  face: "smile",
  timeSlot: "night",    // 追加（任意）: "morning" | "afternoon" | "evening" | "night" | 省略=全時間帯
  rarity: 3,            // 追加（任意）: 1〜5。省略=1
}
```

既存のセリフデータは変更しない（timeSlot省略＝全時間帯、rarity省略＝1として動作）。

**Firestoreスキーマ拡張**:

tasksコレクションの既存フィールドに加え、以下を許可する。

- `rewardPrompt` (string, optional): ご褒美画像生成プロンプト

ユーザー設定用ドキュメント（`users/{uid}`コレクション、1ドキュメント）を新設し、以下を保持する。

- `gachaTickets` (number): ガチャチケット枚数
- `pokeCount` (object): `{ hakone: number, cleade: number, frane: number }` 累計つんつん回数
- `gachaInventory` (array): 獲得済みガチャ景品のID配列

**IndexedDB**:

`task-app-gallery`という名前でDBを開き、以下のストアを作成する。

- `images` ストア: keyPath=`id`。フィールド: `id`, `taskId`(index), `characterId`, `thumbData`(data URL), `fullData`(data URL), `createdAt`

prompt-vault-v2のdb.jsと同等のCRUDユーティリティを実装する。

**週次リセット**:

月曜日にアプリを開いた際、前週（月曜〜日曜）に完了したタスク（`status === 'done'`かつ`completedAt`が前週範囲内）をFirestoreから削除する。リセット判定はlocalStorageに`lastResetWeek`（ISO週番号）を保持し、現在週と比較して実施する。

### §1: URLリンク登録 (F1)

**何を**: ハッシュフラグメントにBase64エンコードしたJSONを含むURLをアプリが解析し、タスク追加フォームにプリフィルする。

**URLフォーマット**:
```
https://task-app-6e764.web.app/#import/<URLセーフBase64>
```

URLセーフBase64: `+`→`-`, `/`→`_`, パディング`=`省略。

**JSONスキーマ**:
```json
{
  "title": "string (必須)",
  "character": "hakone | cleade | frane (必須)",
  "category": "string (任意, CATEGORIESのidに一致)",
  "dueDate": "YYYY-MM-DD (任意)",
  "note": "string (任意)",
  "lines": [
    { "text": "string", "face": "string", "category": "string", "event": "string" }
  ],
  "rewardPrompt": "string (任意)"
}
```

**動作**:
1. App.jsxのマウント時に`location.hash`を検査
2. `#import/`で始まる場合、Base64デコードしてJSONをパース
3. パース成功時: タスク画面を表示し、追加モーダルをプリフィル状態で自動オープン
4. プリフィルされたフィールドは背景色で視覚的に区別する（§B0のキャラカラーのcolorBgを使用）
5. `lines`フィールドが存在する場合: タスク登録時にlines.jsにマージする（Firestoreには保存しない。ローカルのみ）
6. `rewardPrompt`フィールドが存在する場合: Firestoreのタスクドキュメントに保存する
7. ユーザーが「追加」をタップして初めてFirestoreに書き込む。自動登録はしない
8. ハッシュを処理したら`history.replaceState`でURLをクリーンにする

**エラー処理**: デコード失敗・JSON不正時はトーストで通知し、通常のタスク画面を表示する。

### §2: つんつん + キャラ情報 (F2)

**何を**: キャラ画面の個別キャラタブのメインコンテンツ。縦スクロールで、つんつんエリア→キャラ情報→ギャラリー（§7）の順に配置。

**つんつんエリア**:

- 立ち絵表示枠: 最大220×321px（832:1216比率維持）。中央配置
- 画像ソース: `src/assets/characters/{characterId}/default.png`（既存）。将来の表情差分にも対応できるパス設計
- タップ判定: 立ち絵エリア全体。motionの`whileTap={{ scale: 0.95 }}`で視覚フィードバック
- 段階的反応: キャラごとに5段階のセリフを定義。タップ回数に応じて段階が進む。セリフは吹き出し（speech bubble）で立ち絵の下に表示

| 段階 | タップ回数 | ハコネ例 | フラン例 |
|---|---|---|---|
| 1 | 1 | ん？ | 接触を検知 |
| 2 | 3 | もぉ～ | 接触回数が増加しています |
| 3 | 5 | やめてよぉ……♡ | 許容値を超過しています |
| 4 | 10 | ……ひゃん♡ | ……嫌ではありません |
| 5 | 20 | 怒るよ！？ | マスター。 |

- 段階カウンター: 現在段階と次の段階までの残り回数を表示
- セッションカウンター: 今回セッションのタップ回数を表示。アプリ閉じでリセット（sessionStorageまたはReact state）
- 累計カウンター: Firestore `users/{uid}.pokeCount`に永続保存

**キャラ情報カード**:

つんつんエリアの下にdividerを挟んで配置。

- 累計つんつん回数: Firestoreから取得
- 完了タスク数: 全期間の累計（tasksコレクションから`character`一致かつ`status=done`をカウント）
- 得意カテゴリTOP3: 完了タスクのcategory別カウントの上位3件を表示。件数も併記
- ステータス: characters.jsのparams5種をバー表示（baseMood, moodModifier, taskLoyalty, skinshipTolerance, neglectTolerance）。パラメータ値を0-100にスケーリングするか、raw値を表示するかはPG裁量
- 紹介文(bio): characters.jsに`bio`フィールドを追加し、テキストブロックで表示

### §3: ランキング (F3)

**何を**: キャラ画面の📊タブ。今週の完了タスクを3キャラで比較する。

- タイトル: 「今週のランキング」
- 期間表示: 今週の月曜〜日曜の日付
- 棒グラフ: 3キャラを完了数降順にソート。キャラテーマカラーで棒を描画。CSS/SVGで実装（chartsライブラリ不要）
- 合計表示: 今週の全キャラ合計完了数

**weekly champion**:
- 棒グラフの下に、1位のキャラアイコン（既存CharImageコンポーネント、300×300サイズ相当を120×120で表示）を中央配置
- アイコンの下にキャラ名とランキング専用セリフ（1件ランダム抽選）を表示
- 同率1位の場合: characters.jsの配列順で先のキャラをchampionとする

**リセット注記**: 「月曜リセット・完了済みタスクも消去」を画面下部に表示

### §4: ガチャ + アイテム倉庫 (F4)

**何を**: ガチャ画面のメインコンテンツ。ガチャボタン＋チケット表示＋アイテム倉庫。

**チケット**:
- Firestore `users/{uid}.gachaTickets`に保持
- タスク完了時に+1（TaskScreenの完了トグル処理に追加）
- 表示: 画面上部に「チケット N枚」

**ガチャボタン**:
- 円形の大きいボタン（直径110px）。中央配置
- チケット1枚消費でガチャ1回
- マスター権限（常時有効、チケット消費なし）: 「マスター権限: 無制限」と表示

**排出ロジック**:
- 景品データから確率テーブルに基づきランダム抽選
- 確率テーブル: ★1=50%, ★2=30%, ★3=15%, ★4=4%, ★5=1%
- まずレア度を決定し、次にそのレア度の景品プールからランダム1件を選出

**排出演出（M7）**:
- モーダルオーバーレイで表示
- ★1-2: 短い出現アニメーション（motionのscale + opacity）
- ★3: 出現アニメーション＋canvas-confetti（控えめ）
- ★4: 出現アニメーション＋canvas-confetti（花火プリセット）
- ★5: 出現アニメーション＋canvas-confetti（花火＋キャラemoji紙吹雪）＋金色ボーダー
- 演出後タップで閉じる。景品内容を表示

**景品データ**:
ソースコードにバンドルする。構造:
```js
{
  id: "hakone_letter_001",
  character: "hakone",
  rarity: 1,
  title: "ハコネの手紙",
  content: "ショウゴくん、今日もおつかれさま。……えへへ、なんでもない。",
}
```

初期データとして各キャラ×各レア度で最低2件、計30件以上を同梱する。

**アイテム倉庫**:
- ガチャボタンの下にセクション表示
- Firestore `users/{uid}.gachaInventory`に獲得済みアイテムIDを保持
- グリッド表示（3列）。各セルにレア度★表示、キャラemoji、アイテム名
- タップで内容テキストを表示（モーダルまたはインライン展開、PG裁量）

### §5: 時間帯セリフ (F5)

**何を**: セリフ選択ロジックに時間帯条件を追加する。

**時間帯区分**:
| timeSlot | 時間範囲 |
|---|---|
| morning | 05:00 - 10:59 |
| afternoon | 11:00 - 16:59 |
| evening | 17:00 - 22:59 |
| night | 23:00 - 04:59 |

**選択ロジックの変更**:
既存: カテゴリ専用 > generic > フォールバック
拡張: timeSlot一致 > timeSlot省略 の順でフィルタした後、既存のカテゴリ優先ロジックを適用

つまり:
1. イベント一致（register/complete/neglect/cancel）
2. timeSlot一致 OR timeSlot省略のセリフを候補とする
3. カテゴリ専用 > generic > フォールバック
4. rarity確率テーブルで重み付け抽選（§6）

**デバッグ**: 設定画面のデバッグパネルに時間帯手動切替セレクトを追加。「auto」で現在時刻を使用、手動選択で固定。

### §6: レアセリフ演出 (F6)

**何を**: セリフにレア度を設定し、高レア度のセリフ出現時に特別演出を付加する。

**抽選ロジック**:
§5のフィルタ後の候補セリフに対し、rarityフィールドで重み付け抽選を行う。

重みテーブル:
| rarity | 重み | 出現しやすさ |
|---|---|---|
| 1（デフォルト）| 100 | 最も出やすい |
| 2 | 50 | |
| 3 | 20 | |
| 4 | 5 | |
| 5 | 1 | 最も出にくい |

候補にrarity=5のセリフが1件、rarity=1が10件ある場合: rarity=5の出現確率は1/(10×100+1×1)≈0.1%。

**演出**:
- ★1-3: 通常のDialogModal表示（変更なし）
- ★4: モーダルに光彩ボーダー（box-shadow）＋canvas-confetti（短い紙吹雪）
- ★5: モーダルに金色ボーダー＋canvas-confetti（花火＋キャラemoji）＋★5バッジ表示

**デバッグ**: デバッグパネルに「次回セリフをレア度N固定」ドロップダウンを追加。

### §7: ご褒美プロンプト + ギャラリー (F7)

**何を**: タスク完了時にプロンプトをクリップボードコピーする機能と、生成画像を保存・閲覧するギャラリー。

**プロンプトコピー（M4拡張）**:
- タスクの`rewardPrompt`フィールドが存在する場合、完了セリフモーダル（M4）に「📋 プロンプトをコピー」ボタンを表示
- タップでClipboard APIによりテキストをコピー。成功時トースト「コピーしました」、失敗時トースト「コピーに失敗しました」
- Clipboard APIフォールバック: `document.execCommand('copy')`（prompt-vault-v2のcopyText関数と同等）

**ギャラリー（S3内セクション）**:
- キャラ画面の個別キャラタブ最下部に配置
- 3列グリッド。セルの縦横比は832:1216（縦画像基本）
- サムネイル表示: IndexedDBの`thumbData`を使用
- 「＋」セル: タップでファイル選択ダイアログ。accept="image/*"
- 「全て見る >」リンク: ギャラリーを全画面表示（M8）

**画像取り込み**:
- ファイル選択後、以下の処理を実行:
  1. フル画像リサイズ: 長辺800px、JPEG品質75%（prompt-vault-v2のresizeImage関数と同等）
  2. サムネイル生成: 長辺400px、JPEG品質60%（prompt-vault-v2のmakeThumb関数と同等）
  3. IndexedDBに保存: `{ id: crypto.randomUUID(), taskId: null, characterId: activeCharId, thumbData, fullData, createdAt }`
- taskIdは任意。タスク完了時の画像登録フローでは完了したタスクのIDを紐づける

**ギャラリー拡大表示（M8）**:
- サムネイルタップで全画面オーバーレイ。フル画像を表示
- スワイプまたは左右ボタンで前後移動
- 画像ダウンロードボタン（PG裁量）
- タップ or ✕ボタンで閉じる
