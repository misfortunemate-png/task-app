## 作業範囲

Phase 4の全機能を以下の順序で実装する。各ブロックの完了後にコミットすること。

### ブロック①: 基盤（§B0, §B1, §B2）

- 何を: メニュー上部集約、ライブラリ追加、データ構造拡張
- なぜ: 仕様書 §B0, §B1, §B2
- どこで: package.json, App.jsx, MainMenu.jsx, CharTab.jsx, lines.js, characters.js, firestore.rules, 新規ファイル（2層目ナビ、IndexedDB utils）
- 完了条件: メニューが上部固定、2層目が各タブで正しく切替、npm run build 通過

### ブロック②: URLリンク登録（§1）

- 何を: ハッシュフラグメント解析＋フォームプリフィル
- なぜ: 仕様書 §1
- どこで: App.jsx, TaskForm.jsx, 新規ユーティリティ
- 完了条件: テスト#3, #4 パス

### ブロック③: セリフ拡張（§5, §6）

- 何を: 時間帯セリフ＋レアセリフ演出
- なぜ: 仕様書 §5, §6
- どこで: lines.js（データ追加＋選択ロジック変更）, DialogModal.jsx, SettingsScreen.jsx
- 完了条件: テスト#14, #16, #17 パス

### ブロック④: キャラ画面（§2, §3）

- 何を: つんつん＋キャラ情報＋ランキング
- なぜ: 仕様書 §2, §3
- どこで: 新規コンポーネント（キャラ画面、ランキング画面）, App.jsx, firestore.rules
- 完了条件: テスト#5-9 パス

### ブロック⑤: ガチャ（§4）

- 何を: ガチャ画面＋排出演出＋アイテム倉庫
- なぜ: 仕様書 §4
- どこで: 新規コンポーネント, 新規景品データファイル, TaskScreen.jsx（チケット付与）
- 完了条件: テスト#11-15 パス

### ブロック⑥: ギャラリー（§7）

- 何を: プロンプトコピー＋画像取込＋ギャラリー表示

### 最終: 週次リセット＋統合テスト

## 禁止事項

- 仕様書に記載のない機能を追加しない
- 仕様書に記載のないファイルを新規作成しない（ディレクトリ構成の細部はPG裁量）
- 仕様書の原本（docs/spec-phase4.md）を改変しない
- 仕様外の設計判断が必要な場合は作業を停止し報告する
- 既存のPhase 3機能を壊さない（後方互換必須）

## コミットメッセージ形式

```

何を: 実装した内容
なぜ: 仕様書 §N への参照
どのように: 技術的アプローチ
```

例:
```

```

各ブロック・各関数に「何を・なぜ」のコメントを残すこと。仕様書のセクション番号を参照に含めること。

例: `// §2 つんつん段階判定 — 累計をFirestoreに永続化、セッション段階はstateで管理`
